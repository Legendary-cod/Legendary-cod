GIT AND GITHUB NOTES

GIT COMMANDS

-There are different git commands i.e:

  1. git init
  2. git add
  3. git commit m "first commit"
  4. git branch M main
  5. git remote add origin (https://github.com/Allan39543/demo.git)
  6. git push u origin main

git init

What it does:This command initializes a new Git repository in your project. It's like telling Git to start tracking changes in this folder.
When to use it: Use it when you want to start version control for a project that isn’t being tracked by Git yet.
Example;
      ```bash
      git init ```
-After running this, a hidden `.git` folder will be created in your project, and Git starts tracking your files. git add
What it does: This command adds files to the staging area, which is like preparing files for the next commit. You're telling Git, "I want to save the changes I've made in these files."

git add

What it does: This command adds files to the staging area, which is like preparing files for the next commit. You're telling Git, "I want to save the changes I've made in these files."
When to use it: Use it after you've made changes to your files and you’re ready to include them in the next snapshot (commit).
    Example:
      ```bash
      git add index.html ```
    Or to add all files:
      ```bash
      git add .```

-This stages the changes made to `index.html` or all files for the next commit.

git add


What it does: This command shows the current status of your repository. It tells you which files have changes that are staged for commit, which ones have changes that aren't staged yet, and which files are untracked by Git.
When to use it: Use it anytime you want to see what’s going on with your project in terms of Git tracking.
    Example:
      ```bash
      git status ```
-This will display a summary of your current changes, helping you decide what to do next.

git commit

What it does: This command creates a snapshot of the changes in your project. It takes everything in the staging area and stores that snapshot in the history of the repository.
When to use it: Use it after you’ve staged your changes and you’re ready to save a version of your project with a descriptive message.
    Example:
      ```bash
      git commit m "Describe what changes you made" ```

- This commits the staged changes with a message explaining what you did.

git branch -M main

What it does: Renames the current branch to main.
Explanation: By default, Git might call your initial branch master. This command changes its name to main, which is now a common naming convention.

git push

What it does: This command uploads your local repository changes to a remote repository (like on GitHub). It’s like sharing your changes with others or backing them up online.
When to use it: Use it after you’ve committed your changes and you want to send them to a remote repository.
    Example:
      ```bash
      git push origin main ```
- This pushes your commits to the `main` branch of your remote repository called `origin`.
- These commands are the foundation for working with Git, and they help you track and manage your project’s changes efficiently!
